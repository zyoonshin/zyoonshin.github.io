Spring


Spring의 개념
  • 자바 기반의 웹 어플리케이션을 만들 수 있는 프레임워크
  • 스프링 프레임워크는 현대 자바 기반의 엔터프라이즈 어플리케이션을 위한 프로그래밍 및 Configuration Model 제공
  • Java 개발자들은 Spring을 사용하여 웹 서비스를 만들 수 O
  • 수많은 국내 기업과 해외 기업에서 매우 많은 서비스를 만들 때 사용
Spring의 구조

Spring의 특징
  • 자바 객체와 라이브러리들을 관리
  • 톰캣과 같은 WAS가 내장되어 있어 자바 웹 어플리케이션을 구동 할 수 O
  • 경량 컨테이너로 자바 객체를 직접 Spring 안에서 관리
  • 객체의 생성 및 소멸과 같은 생명 주기(Life cycle)을 관리
  • Spring 컨테이너에서 필요한 객체를 가져와 사용
  • 제어의 역전 (IOC, Inversion Of Control)
    • IOC(제어의 역전) : 사용자의 제어권을 다른 주체에게 넘기는 것
    • IOC가 적용된 경우, 객체의 생성을 특별한 관리 위임 주체에게 맡김
    • 사용자는 객체를 직접 생성 X, 객체의 생명주기를 컨트롤하는 주체는 다른 주체가 됨
    • 스프링에게 제어를 위임하여 스프링이 만든 객체를 주입 → 의존성 객체의 메소드 호출
    • 스프링에서는 모든 의존성 객체를 스프링이 실행될때 만들어주고 필요한 곳에 주입
  • 의존성 주입 (DI, Dependency Injection)
    • 객체를 외부(Spring)에서 생성해서 사용하려는 주체 객체(A)에 주입시켜주는 방식
    • 외부(Spring)에서 직접 생성하여 관리하는 경우에는 A와 B의 의존성 줄어듦

Spring Boot


Spring Boot의 개념
  • 스프링(Spring)을 더 쉽게 이용하기 위한 도구
  • 매우 간단하게 프로젝트를 설정할 수 있게 하여, Spring 개발을 조금 더 쉽게 만들어줌
Spring Boot의 구조
