Gradle이란?

  • 그루비(Groovy)를 기반으로 한 빌드 도구
  • Ant와 Maven과 같은 이전 세대 빌드 도구의 단점을 보완하고 장점을 취합하여 만든 오픈소스로 공개된 빌드 도구
Ant
  • XML 기반으로 빌드 스크립트를 작성
  • 자유롭게 빌드 단위 지정
  • 간단하고 사용하기 쉬움
  • 유연하지만 프로젝트가 방대해지는 경우 스크립트 관리나 빌드 과정이 복잡해짐
  • 생명주기(Lifecycle)을 갖지 않아 각각의 결과물에 대한 의존관계 등을 정의해야 함
Maven
  • XML 기반으로 작성
  • 생명주기(Lifecycle)와 프로젝트 객체 모델(POM, Project Object Model)이란 개념 도입
  • Ant의 장황한 빌드 스크립트 개선
  • pom.xml에 필요한 라이브러리를 선언하면 자동으로 해당 프로젝트로 불러와 편리
  • 상대적으로 학습 장벽 ↑
  • 라이브러리가 서로 의존하는 경우 복잡해질 수 O
